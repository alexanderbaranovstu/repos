для работы по регламентным обслуживаниям у нас имеется программный модуль TOService который поддерживает полноценный учет и проверку заказчиком оговоренного объема работ. Модуль предуспатривает создание программной модели объекта с иерархией вида
Объект(Site)	-> Этаж (Floor) -> Область (Area) -> Оборудование (Equipment) -> Сервисные работы 4х типов (Регламентные работы, Аварийные, Модернизация, Гарантийные) -> Еденичная операция (SingleOperation) - перечень операций поясняющих что необходимо сделать в рамках техниеского обслуживания и какой должен быть результат.
Программный модуль содержит ряд отчетов и трекеров которые предназначены для аналитики предполагаемого объема работ, а именно
    countOfEquipment - суммарное количество едениц обслуживаемого в рамках контракта оборудования
    reglamentAction  - суммарное количество регламентных технических обслуживаний на текущий момент времени на объекте
    reglamentLabor   - суммарное количество человекочасов регламентных технических обслуживаний на текущий момент времени на объекте получаемого как сумма всех timeLapse каждого регламентного ТО в рамках контракта
    alarmAction  - суммарное количество аварийных технических обслуживаний на текущий момент времени на объекте
    alarmLabor   - суммарное количество человекочасов аварийных  технических обслуживаний на текущий момент времени на объекте получаемого как сумма всех timeLapse каждого аварийного ТО в рамках контракта	
    upgradeAction  - суммарное количество модернизационных технических обслуживаний на текущий момент времени на объекте
    upgradeLabor   - суммарное количество человекочасов модернизационных технических обслуживаний на текущий момент времени на объекте получаемого как сумма всех timeLapse каждого модернизационного ТО в рамках контракта	
    warantyAction  - суммарное количество гарантийных технических обслуживаний на текущий момент времени на объекте
    warantyLabor   - суммарное количество человекочасов гарантийных технических обслуживаний на текущий момент времени на объекте получаемого как сумма всех timeLapse каждого гарантийного ТО в рамках контракта
Каждое техническое обслуживание оснащено флагами doneFlag и checkedFlag установка которых сообщает системе о том что работы выполнены и проверены соответственно. Конструктив программного модуля TOService предусматривает три профиля пользователей 
- техники (лица которым позволено устанавливать флаги doneFlag)
- менеджеры ( лица которым позволено устанавливать флаги checkedFlag)
- администраторы (лица которые могут устанавливать и снимать все флаги, добавлять этажи, области, оборудование, технические обслуживания, еденичные операции и назначать квоту времени техническому обслуживанию)
Установка флага  doneFlag автоматически перезаписвает поле doneDate на текущую дату и разблокирует для записи поле checkedFlag.
Установка флага  chekedFlag автоматически перезаписвает поле chekedDate на текущую дату.
Вся история операций с установкой флагов и timeLapse доступна в History -> AuditHistory
Для удобства работы с древом проекта предусмотрено несколько табличных видов к которых оттображаются
-все технические обслуживания
-все сделанные и не подписанные заказчиком технические обслуживания
-все не подписанные заказчиком технические обслуживания
Подпись обслуживаний может также осуществляться прямо из таблицы.
Для удобства работы в системе также предусмотрен экспорт данных таблиц в Excel и pdf формат.
