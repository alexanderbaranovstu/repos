                   Модуль ahu предназначен для организации гибкой логики управления различными системами вентиляции.
                   Ключевым классом является BAhuStateMachine. Вся логика работы построена на том что этот модуль стоит во главе иерархии. Все остальные классы 
                   EquipmentFMode, EquipmentFWinter, MixCamera, Cooler, Heater, NumericValue, BooleanValue, EnumValue, StringValue ДОЛЖНЫ быть прямыми потомками 
                   под классом BAhuStateMachine.
                   BAhuStateMachine содержит свойство Mode (AlarmRw, Alarm, Standby, On). Компоненты BEquipmentFMode, EquipmentFWinter, MixCamera, Cooler, Heater, NumericValue, 
                   BooleanValue, EnumValue, StringValue из которых строятся программные узлы вентагрегата - приточный вентилятор, камера смешения, нагреватель, и.т.п
                   следят за состоянием Mode и переключают свой выход Out в предопределенное состояние согласно логике (AlarmRw, Alarm, Standby, On). 
                   Коммоненты EquipmentFMode, EquipmentFWinter, NumericValue, BooleanValue, EnumValue, StringValue могут быть оснащены аварийными расширениями
                   которые в зависимости от того насколько критична авария - могут переводить систему в режим AlarmRw, Alarm либо вообще оставить систему в работе без изменений
                   для этого предназначены поля sendAlarmRw и sendAlarm - и в зависимости от того как установлены переключатели "SendToParent", "DoNothig"
                   
BAhuStateMachine    - Переключение осуществляется между состояниями AlarmRw, Alarm, Standby, On. 
                    В  класс встроены методы setAlarm(), resetAlarm(), setAlarmRw(), resetAlarmRw()
                    для того что бы сторонние классы могли переводить BAhuStateMachine в состояние 
                    AlarmRw или Alarm. Предпологается что переключение между  Standby и On будет
                    осуществляться посредством физических входов или планировщика
BAhuState           - Енумерация AlarmRw, Alarm, Standby, On
BEquipmentFMode     - Класс который распологается под BAhuStateMachine, смотрит за свойством Mode, 
                    и переключает свой выход out в предопределенное в самом классе положение выхода
                    в зависимости от состояния Mode. У класса есть возможность вызывать методы
                    родительского класса  setAlarm(),  setAlarmRw(), посредством установки флагов
                    sendAlarmRw, sendAlarm в положение SendToParent. Для того что бы авария от 
                    класса могла влиять на состояние родительского класса BAhuStateMachine необходимо 
                    добавить к BEquipmentFMode стандартное расширение BooleanCommandFailureAlarmExt
                    и его корректно настроить. Предполагается что имя класса будет соответствовать 
                    оборудованию его представляющему. Например SupplyFan, AlarmLamp, RotorExchanger.....
BEquipmentFWinter   - То же что и BEquipmentFMode только смотрит класс не за свойством Mode, а за Winter, таким 
                    образом определяя состояние переменной Out как функцию от Winter. Так же как в классе
                    BEquipmentFMode,  У класса есть возможность вызывать методы
                    родительского класса  setAlarm(),  setAlarmRw(), посредством установки флагов
                    sendAlarmRw, sendAlarm в положение SendToParent. 
BMixCamera          -Класс позволяющий оценить температуру воздушной смеси в камере смешения на основании TempOut, FlowSup, TempRet, FlowRet и процента открытия КС. Результатом работы функции
                    является положение заслонки в камере смешения SptMixCamera. Класс учитывает режим работы машины (AlarmRw, Alarm, Standby, On) для определения положения выхода используются
                    константы   enum BPos {    range    {      nullPosition,      zeroPosition,      minPosition,      maxPosition,      holdPosition,      regulator    }  }
                    Алгоритм построен так что при повышении уличной температуры TempOut выше TempRet запускается механизм сохранения холода в помещении и заслонка перекрывает более горячий приток
                    ограничивае его минимальным значением которое также конфигурируется при параметрировании класса.
BCooler             - Класс который можно использовать для управления охладителем. Оснащен PID регулятором и переключателем выхода в зависимости от (AlarmRw, Alarm, Standby, On)
BHeater             - Класс для водяного нагревателя. Имеет встроенную защиту от размораживания и оснащен отключаемым каскадным регулятором. Температура поддержания Т2 в режиме Standby = 25 °С
BNumericValue       - Абстрактное аналоговое значение для которого можно определить аварийные диапазоны
                    при помощи стандартных alarmExtentions и предусмотреть трансляцию аварий вверх
                    к машине состояний. Так например можно определить минимальную или максимальную температуру
                    притока, обратной воды, обрыва датчиков, коммуникационной ощибки с модулем ввода вывода.
                    У класса есть возможность вызывать методы родительского класса  setAlarm(),  setAlarmRw(), посредством установки флагов
                    sendAlarmRw, sendAlarm в положение SendToParent. 
BBooleanValue       - То же что и BNumericValue только типа Boolean          
BEnumnValue         - То же что и BNumericValue только типа Enum  
BStringValue        - То же что и BNumericValue только типа String
